# Folo 沉淀

记录从 Folo 时间线里筛出来、值得长期保留的知识。只放可复用的工作流、判断标准和产品变化，不做纯新闻堆砌。

## 2026-06-13

### 1. 先结构化输入，再让 AI 执行固定流程

Tw93 的简历生成思路不是“让 AI 自由发挥”，而是先把原始素材整理成 `md`，再用固定命令触发产出，最后只做少量微调。

可沉淀成一句话：

- `结构化输入 + 固定指令 + 少量修正`，比临时对话更稳定。

原文来源：

- [HiTw93 / Kami 简历工作流](https://x.com/HiTw93/status/2065724271486493084)

### 2. 做 Agent Skill 之前，先验证它值不值得存在

“鲁班”这条的价值不在于润色，而在于一个更强的流程：先联网对标，再用结构、实测、活体三把尺打分，最后才决定怎么改。

可沉淀成一个判断模板：

- 先问 `这个 Skill 是否必要`
- 再问 `它和同类相比差在哪里`
- 最后才进入改写和优化

原文来源：

- [Geek / 鲁班 Skill](https://x.com/geekbb/status/2065688116246278429)

### 3. 免费层配额会变，资源组织要按扩展性设计

Vercel Hobby Blob store 从 5 提到 100，但存储、操作和传输限制仍然存在。真正可复用的不是“额度变大了”，而是资源组织方式。

可沉淀成：

- 不要把免费额度当成长期容量规划
- 按 `项目 / 环境 / 区域` 拆分资源更稳
- 只看数量不看操作与传输限制，容易误判可用性

原文来源：

- [Vercel Changelog / Increased Blob store limit for Hobby users](https://vercel.com/changelog/increased-blob-store-limit-for-hobby-users)
