# 模型依赖风险

来源：Anthropic 关于 `Fable 5` / `Mythos 5` 下线的公告。

把大模型当成外部依赖时，风险不是只有“贵不贵”，还包括“能不能用、谁能用、什么时候不能用”。

## 主要风险

- 地区或身份限制突然生效
- 模型被临时下架
- 额度策略突然变化
- 价格或套餐结构变化
- 供应商因为合规、政策或安全原因调整能力

## 对应用的影响

- 线上功能可能直接失效
- 依赖单一模型的产品会被动停摆
- 业务逻辑和提示词会和模型强绑定
- 用户体验会被供应商策略牵着走

## 规避方式

- 保留多模型切换能力
- 把关键能力做成 provider 抽象
- 不把“某一个模型”写死进业务主流程
- 提前准备降级方案和 fallback
- 对外部模型做可替换性设计

## 结论

- 大模型不是永久基础设施
- 模型能力、价格和可用性都要按供应链管理
- 重要业务不要只押一个供应商
